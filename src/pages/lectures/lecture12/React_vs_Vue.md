# React vs. Vue: 프레임워크 비교

React와 Vue는 오늘날 가장 인기 있는 프론트엔드 JavaScript 라이브러리/프레임워크입니다. 둘 다 컴포넌트 기반 아키텍처를 채택하여 재사용 가능하고 관리하기 쉬운 UI를 구축하는 데 중점을 둡니다. 하지만 철학, 문법, 생태계 등에서 몇 가지 중요한 차이점을 가집니다.

---

### 1. 핵심 철학

- **React**: "UI를 만들기 위한 JavaScript 라이브러리"를 표방합니다. 상태 관리, 라우팅 등 UI 이외의 기능은 공식적으로 지원하지 않고, 서드파티 라이브러리(예: Redux, React Router)와 함께 사용하는 것을 권장합니다. 유연성이 높고 JavaScript 중심적인 개발을 지향합니다.

- **Vue**: "프로그레시브 프레임워크(Progressive Framework)"를 표방합니다. 핵심 라이브러리는 React처럼 UI 렌더링에 집중하지만, 상태 관리(Pinia), 라우팅(Vue Router) 등 필수적인 기능들을 공식 라이브러리로 제공하여 하나의 완성된 프레임워크 생태계를 구축합니다.

### 2. 템플릿 및 문법

- **React**: **JSX (JavaScript XML)**를 사용합니다. HTML 구조와 JavaScript 로직을 하나의 파일 안에 함께 작성하는 방식입니다. JavaScript의 모든 기능을 활용할 수 있어 동적인 UI를 만드는 데 강력하지만, 초심자에게는 익숙해지는 데 시간이 걸릴 수 있습니다.

  ```jsx
  function Greeting({ name }) {
    return <div className="greeting">{name ? <h1>Hello, {name}!</h1> : <p>Guest</p>}</div>;
  }
  ```

- **Vue**: **HTML 기반 템플릿**을 사용합니다. 기존 HTML 문법과 매우 유사하며, `v-if`, `v-for` 같은 디렉티브를 사용하여 동적인 로직을 추가합니다. 웹 개발에 익숙한 사람이라면 매우 직관적으로 느낄 수 있습니다. (물론 Vue도 JSX를 사용할 수 있습니다.)

  ```html
  <template>
    <div class="greeting">
      <h1 v-if="name">Hello, {{ name }}!</h1>
      <p v-else>Guest</p>
    </div>
  </template>

  <script setup>
    defineProps({ name: String });
  </script>
  ```

### 3. 상태 관리

- **React**: 컴포넌트 내 지역 상태는 `useState`, `useReducer` 훅을 사용합니다. 전역 상태 관리를 위해서는 `Context API`를 사용하거나, 더 복잡한 애플리케이션에서는 `Redux`, `Zustand`, `Recoil` 같은 외부 라이브러리를 주로 사용합니다.

- **Vue**: Vue 3부터는 **Composition API** (`ref`, `reactive`)를 통해 강력한 반응형 상태 관리를 기본으로 제공합니다. 전역 상태 관리가 필요할 경우 공식 라이브러리인 **Pinia** (과거엔 Vuex)를 사용하여 예측 가능하고 관리하기 쉬운 상태 관리를 할 수 있습니다.

### 4. 반응성(Reactivity)

- **React**: **불변성(Immutability)**을 기반으로 작동합니다. 상태가 변경되면 `setState`와 같은 함수를 통해 새로운 상태 객체나 배열을 생성하여 교체합니다. React는 가상돔(Virtual DOM)을 통해 이전 상태와 현재 상태를 비교하여 변경된 부분만 실제 DOM에 업데이트합니다.

- **Vue**: **변경 가능성(Mutability)**을 기반으로 합니다. Vue 3의 Composition API는 JavaScript 객체를 `Proxy` 객체로 감싸서 데이터의 변경을 직접 감지합니다. 데이터가 변경되면 해당 데이터를 사용하는 모든 곳에 자동으로 업데이트를 전파합니다. 이 방식은 개발자가 상태를 직접 수정하는 것처럼 코드를 작성할 수 있어 더 직관적일 수 있습니다.

### 5. 러닝 커브 (학습 곡선)

- **React**: JSX 문법과 JavaScript 중심의 개발 방식, 상태 관리의 불변성 개념 등 초기에 학습해야 할 내용이 상대적으로 많을 수 있습니다. 하지만 JavaScript에 익숙하다면 빠르게 적응할 수 있습니다.

- **Vue**: HTML과 유사한 템플릿 문법과 잘 정리된 공식 문서 덕분에 일반적으로 React보다 러닝 커브가 낮다고 평가받습니다. 프레임워크가 제공하는 가이드라인이 명확하여 초심자가 시작하기에 좋습니다.

### 6. 생태계 및 커뮤니티

- **React**: Facebook(Meta)의 지원을 받으며, 훨씬 더 큰 커뮤니티와 방대한 서드파티 라이브러리 생태계를 가지고 있습니다. 거의 모든 종류의 UI 컴포넌트와 유틸리티를 찾을 수 있습니다.

- **Vue**: React만큼은 아니지만 매우 활발하고 강력한 커뮤니티를 보유하고 있습니다. 특히 중국 시장에서 압도적인 인기를 누리고 있으며, 공식적으로 지원하는 라이브러리들의 완성도가 높습니다.

---

### 요약 비교표

| 구분          | React                                        | Vue                                   |
| :------------ | :------------------------------------------- | :------------------------------------ |
| **개발 주체** | Meta (Facebook)                              | Evan You (개인) 및 커뮤니티           |
| **핵심 철학** | JavaScript 라이브러리 (유연성)               | 프로그레시브 프레임워크 (점진적 채택) |
| **템플릿**    | JSX (JavaScript + HTML)                      | HTML 기반 템플릿 (디렉티브 사용)      |
| **상태 관리** | `useState`, `Context API`, Redux, Zustand 등 | `ref`, `reactive`, Pinia (공식)       |
| **반응성**    | 불변성 기반, 가상돔 비교                     | 변경 가능성 기반, Proxy 객체          |
| **러닝 커브** | 상대적으로 높음                              | 상대적으로 낮음                       |
| **생태계**    | 매우 거대하고 방대함                         | 크고 활발하며, 공식 지원이 탄탄함     |

### 결론

어떤 것을 선택할지는 프로젝트의 요구사항, 팀의 기술 스택, 그리고 개발자의 개인적인 선호도에 따라 달라집니다.

- **React**는 대규모 애플리케이션, 풍부한 생태계 활용, 그리고 JavaScript 중심의 유연한 개발을 선호하는 경우에 좋은 선택이 될 수 있습니다.
- **Vue**는 빠른 프로토타이핑, 명확한 가이드라인, 그리고 HTML/CSS에 더 익숙한 개발자들이 팀에 많을 경우 더 나은 생산성을 보여줄 수 있습니다.
